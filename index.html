Bienvenue 👋

Cet onglet rassemble les actions courantes du cycle d'une saison. Chaque sous-onglet est une tâche concrète, avec quand l'effectuer, ce qu'il faut sous la main, et la marche à suivre étape par étape.

Avant tout : se connecter à WordPress

Quasiment chaque tâche de cet onglet commence par "se connecter à WordPress". Procédure rapide, avec un détour obligatoire par un bug récurrent :

  1. Récupérer les identifiants dans le coffre LastPass (entrée "Elementor — site CUG").
  2. Ouvrir cug38.fr/wp-admin, cliquer le bouton violet Log in with Elementor.
  3. Saisir email + mot de passe Elementor. (Pas Continue with Google ni SSO.)
  4. Si demandé : code 2FA reçu par e-mail sur orga@cug38.fr.
  5. Erreur "Something went wrong" → c'est normal, fermer l'onglet.
  6. Re-cliquer Log in with Elementor → tableau de bord.

Procédure détaillée avec screenshots : Le contenu → Connexion & admin.

Inscriptions à chaque rentrée

À chaque rentrée d'une nouvelle saison, l'article "Les inscriptions sont ouvertes" doit revenir en tête de la page d'accueil. Pas besoin d'en créer un nouveau — on republie celui qui dort déjà dans la base.

Le principe Le formulaire d'inscription est le même d'une saison à l'autre (il vit dans Google Forms côté orga@cug38.fr). L'article "Les inscriptions sont ouvertes" existe déjà dans les Actualités. Pour le faire remonter sur la home : on change sa date de publication.

Pas-à-pas

  1. Se connecter à WordPress.
  2. Menu de gauche : Articles → Tous les articles, filtrer par catégorie Actualités.
  3. Trouver l'article "Les inscriptions sont ouvertes" (vérifier le titre exact dans la liste).
  4. Cliquer Modifier.
  5. Optionnel : ajuster le titre et le contenu si la formulation évolue.
  6. Sidebar droite : section Publication → cliquer sur la date de publication → mettre la date d'aujourd'hui.
  7. Cliquer Mettre à jour. L'article remonte automatiquement en tête de la home.
Si l'article n'apparaît pas tout de suite côté public Le site garde un cache des pages publiques. Aller sur la page concernée (home, /billetterie/, Espace choristes…), cliquer Clear Posts Cache dans le bandeau noir d'admin tout en haut, recharger. Détails : Le contenu → Les articles → Si un article ne s'affiche pas.
Où le résultat apparaît Les 2 cards du haut de la home : "Les inscriptions sont ouvertes" doit apparaître dans l'une d'elles (selon les autres actualités plus récentes).

Voir aussi : structure d'un article Actualités dans Le contenu → Les articles → Actualités.

AvancéPourquoi le formulaire reste le même

Avant, le formulaire Google Forms contenait des champs liés à la saison spécifique (ex. "Saison 2024-2025"). À chaque rentrée, il fallait créer un nouveau formulaire et changer l'iframe sur la page Infos pratiques. Maintenant le formulaire est rédigé pour être réutilisable d'une saison à l'autre — donc on ne touche plus ni au formulaire, ni à l'iframe sur la page. Pour modifier les questions : ouvrir directement le Google Form dans Drive côté orga@cug38.fr.

Mot de passe choristes à chaque rentrée

Le mot de passe de l'Espace choristes est unique et partagé entre tous les choristes via le livret de partitions distribué à la rentrée. Il se change une fois par an. Le réglage vit dans l'extension Password Protect WordPress sur la page Espace Choristes.

Subtilité : ouvrir avec Gutenberg, pas Elementor Le formulaire de mot de passe est dans la sidebar de l'éditeur WordPress (Gutenberg). Si on ouvre la page avec Elementor, l'option n'apparaît nulle part — c'est ce qui fait souvent perdre 10 minutes.

Pas-à-pas

  1. Se connecter à WordPress.
  2. Menu de gauche : Pages → Toutes les pages.
  3. Sur la ligne "Espace Choristes", cliquer directement sur le titre (PAS sur Modifier avec Elementor). L'éditeur Gutenberg s'ouvre.
  4. Sidebar de droite : descendre jusqu'à la section Password Protect WordPress.
  5. Cliquer "Edit" à côté de "Password protected by 1 role".
  6. Le rôle est Global. Le champ Passwords contient le mot de passe actuel.
  7. Remplacer la valeur du champ Passwords par le nouveau mot de passe.
  8. Cliquer Submit pour appliquer.
  9. Cliquer Enregistrer en haut à droite de la page WordPress pour sauvegarder définitivement.
  10. Mettre à jour l'entrée LastPass CUG — Espace choriste avec le nouveau mot de passe.
  11. Diffuser le nouveau mot de passe aux choristes (livret de partitions, email).
Côté admin : la sidebar Gutenberg de la page Espace Choristes avec le formulaire Password Protect WordPress. C'est ici qu'on remplace le mot de passe.
Où le résultat apparaît Écran d'accès à l'Espace choristes (cug38.fr/espace-choristes/). Les choristes actuels seront déconnectés et devront ressaisir le nouveau mot de passe à leur prochaine visite.
Côté choriste : l'écran d'accès à cug38.fr/espace-choristes/, avec ses instructions (« Entrez le mot de passe fourni dans votre livret de partitions… ») + un visuel du livret.

Voir aussi : Le contenu → Les pages — Plus de détails sur l'éditeur de pages.

Nouveau CA après chaque AG

À chaque renouvellement du Conseil d'Administration (typiquement après l'assemblée générale annuelle), le trombinoscope affiché sur la page Le chœur et dans l'Espace choristes → Les news du CA doit refléter la nouvelle composition. Tout passe par les articles — aucune intervention dans Elementor.

Le principe Un membre du CA = un article (catégorie CA). Le titre = le rôle. L'extrait = le nom. L'image à la une = la photo. Le champ Ordre détermine la position dans la grille.

Ce qu'il faut sous la main

  • La liste des nouveaux membres avec leur rôle et leur nom complet.
  • Une photo carrée par membre (la grille attend un format carré pour un rendu uniforme).
  • Les identifiants WordPress (cf. Bienvenue → Se connecter).

Pas-à-pas

  1. Se connecter à WordPress.
  2. Menu de gauche : Articles → Tous les articles. Filtrer par catégorie CA.
  3. Pour chaque membre qui part : passer son article en brouillon (au survol de la ligne, lien Modification rapide → Statut Brouillon → Mettre à jour). On garde en brouillon plutôt que poubelle, au cas où.
  4. Pour chaque nouveau membre : créer un nouvel article (ou dupliquer un existant comme modèle), puis remplir :
    • Titre = le rôle (ex. « Président », « Trésorière adjointe », « Membre »).
    • Extrait (sidebar droite) = le nom complet (ex. « Marie Dupont »).
    • Image à la une (sidebar droite) = la photo, format carré.
    • Attributs de page → Ordre (sidebar droite) = un nombre. Les valeurs actuelles vont par pas de 10 (Président = 10, Présidente adjointe = 20, etc.). Plus petit = remonte dans la grille.
    • Catégorie : cocher CA uniquement.
  5. Cliquer Publier.
  6. Vérifier sur la page publique cug38.fr/le-choeur/, section Le CA.
Si l'article n'apparaît pas tout de suite côté public Le site garde un cache des pages publiques. Aller sur la page concernée (home, /billetterie/, Espace choristes…), cliquer Clear Posts Cache dans le bandeau noir d'admin tout en haut, recharger. Détails : Le contenu → Les articles → Si un article ne s'affiche pas.
Où le résultat apparaît Page /le-choeur/ (section « Le CA — Une équipe au top ») et Espace choristes → onglet Les news du CA.
Section Le CA — Une équipe au top sur la page /le-choeur/. Une card par article catégorie CA, triées par le champ Ordre.
AvancéComment ça marche techniquement

La section CA utilise un widget Elementor Loop Grid filtré sur la catégorie CA, trié par menu_order croissant. Le champ « Ordre » est exposé dans Gutenberg grâce au snippet CUG — Exposer le champ Ordre. Quand on publie / dépublie un article CA, la grille se met à jour automatiquement sans toucher au template.

Voir aussi : Le contenu → Les articles → CA — Structure d'un article CA (champs, ordre).

Nouveau programme chaque semestre

À chaque nouveau semestre (typiquement deux fois par an), l'onglet Le programme de l'Espace choristes doit refléter le nouveau répertoire et les nouveaux fichiers de travail. Le précédent bascule automatiquement dans Anciens programmes.

Le principe Un semestre = un article (catégorie Programme). Le plus récent (par date de publication) = "courant" ; les autres = "Anciens". Quatre champs personnalisés pointent vers les fichiers de travail.

Ce qu'il faut sous la main

  • Le dossier MuseScore sur Drive (lien partageable).
  • L'ID de playlist YouTube du semestre (juste la partie après list=).
  • Le livret PDF sur Drive — pas IMSLP, pas ailleurs, sur Drive.
  • Le dossier MP3s par pupitre sur Drive.

Pour préparer ces fichiers sur Drive (où les ranger, comment partager) : L'infrastructure → Google Drive.

Pas-à-pas

  1. Préparer les fichiers sur Drive (orga@cug38.fr) → Fichiers site web → Programme → [nom du semestre]. Récupérer les 4 URLs.
  2. Se connecter à WordPress.
  3. Articles → Ajouter un article.
  4. Remplir :
    • Titre = nom du semestre (ex. « Programme 2026/2027 — Semestre 1 »).
    • Contenu (Gutenberg) = un bloc Titre H4 (nom de l'œuvre) + un paragraphe descriptif (compositeur, partenaires, lieu, dates).
    • Catégorie : cocher Programme uniquement.
    • Pas d'extrait, pas d'image à la une — le template ne les utilise pas pour cette catégorie.
  5. Sidebar droite → Champs personnalisés :
    • programme_youtube_playlist = l'ID de la playlist.
    • programme_musescore = l'URL du dossier MuseScore.
    • programme_livret_pdf = l'URL du livret PDF (Drive).
    • programme_mp3s = l'URL du dossier MP3s.

    Champs vides = sous-cartes correspondantes cachées automatiquement.

  6. Cliquer Publier.
  7. Vérifier sur l'Espace choristes → onglet Le programme. L'ancien semestre doit avoir basculé dans Anciens programmes.
Si l'article n'apparaît pas tout de suite côté public Le site garde un cache des pages publiques. Aller sur la page concernée (home, /billetterie/, Espace choristes…), cliquer Clear Posts Cache dans le bandeau noir d'admin tout en haut, recharger. Détails : Le contenu → Les articles → Si un article ne s'affiche pas.
Où le résultat apparaît Espace choristes → onglet Le programme (le nouveau) et Anciens programmes (les autres).
L'onglet Le programme de l'Espace choristes. Tout (titre, descriptif, sous-cartes MuseScore / Livret / YouTube) est piloté par l'article Programme le plus récent.
AvancéMettre deux programmes en cours côte à côte

Cas particulier : si deux semestres tournent en parallèle (ex. un programme principal + un programme récurrent comme Conférence ou Cérémonies), on peut afficher les deux dans l'onglet Le programme et décaler l'archive d'un cran.

Côté onglet "Le programme" (Espace choristes) :

  1. Ouvrir la page Espace Choristes dans Elementor.
  2. Sélectionner la Grille de boucle de l'onglet Le programme.
  3. Panneau de gauche, onglet Contenu → Query : passer Posts Per Page de 1 à 2.
  4. Enregistrer. Les 2 programmes les plus récents s'affichent côte à côte.

Côté onglet "Anciens programmes" (pour exclure les 2 affichés en courant) :

  1. Sélectionner la Grille de boucle de l'onglet Anciens programmes.
  2. Panneau de gauche, onglet Contenu → Query, section Exclude → mettre Offset à 2 (au lieu de 1 par défaut).
  3. Enregistrer. La grille saute les 2 premiers articles (déjà affichés en courant) et liste à partir du 3e.

Pour revenir à un seul programme en cours : repasser Posts Per Page à 1 et Offset à 1.

AvancéComment ça marche techniquement

Une Grille de boucle Elementor sur le template Programme (post id 3073) lit les articles catégorie Programme et trie par date DESC. La première card = le programme courant ; les suivantes vont dans la grille « Anciens ». Les sous-cartes (MuseScore / YouTube / livret / MP3s) utilisent le pattern cug-cond / cug-cond-host avec les shortcodes [programme_musescore], [programme_livret_pdf], etc. — voir Catalogue des snippets.

Voir aussi : Le contenu → Les articles → Programme — Structure d'un article Programme (4 champs personnalisés, shortcodes).

Ouverture de la billetterie quelques semaines avant chaque concert

Quand la billetterie d'un concert ouvre, deux articles entrent en scène : (1) l'article Billetterie qui pilote la page /billetterie/ et la card billetterie de la home, et (2) une Actualité "Billetterie ouverte" qui annonce l'ouverture et apparaît dans les cards du haut de la home.

Le principe Un article catégorie Billetterie contient tout le concert (affiche, description, boutons HelloAsso, lieu via custom field). En parallèle, une Actualité annonce l'ouverture des ventes.

Étape 1 — Créer l'article Billetterie

Ce qu'il faut sous la main

  • L'affiche du concert.
  • La description du programme.
  • Les liens HelloAsso de la billetterie (un par session).
  • Le nom du lieu tel qu'on veut le voir sur la carte (ex. « Espace Bergès Villard-Bonnot »).
  • La date de fin souhaitée — typiquement le lendemain du dernier concert (format YYYY-MM-DD).

Pas-à-pas

  1. Se connecter à WordPress.
  2. Articles → Ajouter un article.
  3. Remplir :
    • Titre = nom court du concert (ex. « Materia Symphony »).
    • Extrait = sous-titre (ex. « et autres musiques de jeux vidéo »).
    • Image à la une = l'affiche.
    • Contenu (Gutenberg) = paragraphes descriptifs + bloc de boutons HelloAsso (un bloc Boutons, un bouton par session avec son URL HelloAsso).
    • Catégorie : cocher Billetterie uniquement.
  4. Sidebar droite → Champs personnalisés :
    • date_fin = la date jusqu'à laquelle la campagne reste visible (YYYY-MM-DD, en général le lendemain du dernier concert).
    • event_map = le nom du lieu. La carte Google Maps se construit toute seule.
  5. Cliquer Publier.

Étape 2 — Créer l'Actualité "Billetterie ouverte"

Pas-à-pas

  1. Toujours dans le contenu : Articles → Ajouter un article.
  2. Remplir :
    • Titre = court et accrocheur (ex. « Billetterie ouverte pour Materia Symphony »).
    • Contenu (Gutenberg) = un paragraphe d'annonce + un lien « Plus d'informations ici » qui pointe vers la page Billetterie.
    • Image à la une = format paysage (l'affiche recadrée si besoin — sinon recadrage moche sur la card).
    • Extrait = 1-2 lignes — c'est ce texte qui s'affiche sur la card de la home. À ne pas oublier.
    • Catégorie : cocher Actualités uniquement.
  3. Cliquer Publier.
  4. Vérifier la home : la card 1 doit afficher cette nouvelle actualité, et la card billetterie doit afficher le nouveau concert.

Étape 3 — Contacter les inscrits à la newsletter

Pas-à-pas

  1. Se connecter à Drive avec orga@cug38.fr.
  2. Ouvrir le Sheet Newsletter dans My Drive → Fichiers site web. Il contient la liste consolidée des adresses inscrites via le formulaire S'inscrire de la page Billetterie (alimenté automatiquement par Apps Script — voir L'infrastructure → Google Drive pour le détail du pipeline).
  3. Copier la colonne A (les emails). En général on a entre quelques dizaines et quelques centaines d'adresses.
  4. Composer le message depuis Gmail orga@cug38.fr (ou Mailchimp / Brevo si on préfère) : sujet court, lien vers la page Billetterie, mention des dates et du lieu, lien vers HelloAsso.
  5. Coller les adresses en Cci (et pas dans le champ À) — un seul envoi de masse, sans exposer les adresses des destinataires les unes aux autres.
  6. Mettre orga@cug38.fr dans le champ À (l'envoi à soi-même + Cci = pratique courante pour les newsletters associatives).
  7. Envoyer.
Toujours en Cci, jamais en À Mettre les adresses des inscrits en À ou en Cc exposerait leurs emails à tous les destinataires — manquement RGPD évident. Le champ Cci (copie carbone invisible) cache les destinataires les uns des autres. À vérifier à chaque envoi.
Où le résultat apparaît Article Billetterie → page /billetterie/ + card billetterie sur la home (tant que date_fin n'est pas passée). Actualité « Billetterie ouverte » → cards 1 ou 2 du haut de la home. Email newsletter → dans les boîtes des inscrits, avec un retour direct vers la page Billetterie.
La page /billetterie/ en saison : l'affiche, la description, les boutons HelloAsso, la carte du lieu — tout vient de l'article Billetterie actif.
AvancéComment ça marche techniquement

La page Billetterie contient une Grille de boucle filtrée par le snippet billetterie_active (articles catégorie Billetterie avec date_fin ≥ aujourd'hui). Si aucun article n'est actif, un placeholder « pas encore ouverte » s'affiche à la place, grâce au shortcode sentinelle [no_active_billetterie] + le pattern cug-cond / cug-cond-host. La carte du lieu est rendue par le shortcode [event_map_embed] qui lit le champ event_map. Catalogue snippets : Catalogue des snippets.

Voir aussi : Le contenu → Les articles → Billetterie — Structure d'un article Billetterie (date_fin, event_map, etc.).

Archivage du concert dans les jours qui suivent le concert

Après un concert, le travail principal est de créer une fiche Projet qui ira rejoindre l'archive sur la page Nos concerts et remontera sur la home (3e card Notre dernier concert). C'est elle qui rassemble vidéo, affiche, partenaires, programme musical.

Note : la billetterie disparaît toute seule Quand la date_fin de l'article Billetterie est dépassée, l'article bascule automatiquement hors de la page Billetterie et de la home. Rien à faire pour la retirer.

Créer la fiche Projet

Ce qu'il faut sous la main

  • L'affiche du concert.
  • Le résumé court (dates, programme en un mot) pour l'extrait.
  • L'URL YouTube de la captation si elle est déjà en ligne (sinon on l'ajoutera plus tard).
  • Les logos des partenaires (Crous, CVEC, Ville, etc.).
  • Le texte descriptif du programme musical.

Pas-à-pas

  1. Se connecter à WordPress.
  2. Articles → Ajouter un article.
  3. Remplir les méta-données dans l'éditeur Gutenberg :
    • Titre = nom du concert (sert de titre de card + H1 de la fiche détail).
    • Extrait (sidebar droite) = court résumé (dates, programme). Visible sur la card de la grille.
    • Image à la une (sidebar droite) = l'affiche, format portrait (verticale sur la card de la grille).
    • Catégorie : cocher Projets uniquement.
  4. Cliquer Publier (premier passage pour fixer slug, image, extrait).
  5. Cliquer le bouton bleu Modifier avec Elementor en haut à gauche pour basculer en édition visuelle.
  6. Construire le contenu de la fiche dans Elementor — le plus simple est de dupliquer la fiche d'un concert précédent et de tout remplacer :
    • Vidéo YouTube embarquée (widget Vidéo).
    • Affiche du concert.
    • Section « Au programme » avec texte descriptif.
    • Logos partenaires (Crous, CVEC, Ville, etc.).
  7. Mettre à jour. Vérifier sur la home (3e card) et sur la page Nos concerts.
Ne toucher qu'au contenu central Dans l'éditeur Elementor d'un Projet, on voit aussi le titre, l'image à la une, la sidebar « Autres projets », le header, le footer. Ne PAS y toucher — ces éléments sont rendus par le template Projet (Article unique) mutualisé entre toutes les fiches.
Où le résultat apparaît Home (3e card « Notre dernier concert » — le dernier Projet publié remonte automatiquement) + page /nos-concerts/ qui liste tous les Projets.
Vue détaillée d'une fiche Projet ouverte depuis /nos-concerts/ : titre, date, affiche, description, lien vidéo — tout vient de l'article Projets construit dans Elementor.

Voir aussi : Le contenu → Les articles → Projets — Structure d'une fiche Projet + pièges Elementor.

Captations et photos quand la vidéo et les photos sont prêtes

Quand la captation vidéo d'un concert est mise en ligne sur la chaîne YouTube du CUG, on crée un article Youtube qui apparaît sur la 4e card de la home. En parallèle, les photos du concert sont rangées dans le Drive pour alimenter l'Espace choristes.

La captation vidéo

Le principe Une captation = un article (catégorie Youtube) + un ajout à la playlist Captations de la chaîne YouTube du CUG. Le champ personnalisé youtube-link contient l'URL — la card de la home embarque la vidéo automatiquement.

Ce qu'il faut sous la main

  • L'URL YouTube complète de la vidéo (https://www.youtube.com/watch?v=XXXX).
  • Le titre du concert / de la captation.
  • Un court texte descriptif du concert.

Pas-à-pas

  1. Sur YouTube (chaîne du CUG, compte orga@cug38.fr) : ajouter la vidéo à la playlist Captations → la playlist est ici. Clic sur les trois points (···) de la vidéo → Enregistrer dans la playlist → cocher Captations.
  2. Se connecter à WordPress WordPress.
  3. Articles → Ajouter un article.
  4. Remplir :
    • Titre = titre du concert / de la captation. Obligatoire.
    • Contenu (Gutenberg) = un court paragraphe descriptif. Obligatoire.
    • Catégorie : cocher Youtube uniquement.
    • Pas d'image à la une, pas de vignette à ajouter — la card de la home affiche directement la vidéo YouTube.
  5. Sidebar droite → Champs personnalisés : youtube-link = l'URL YouTube complète. Indispensable — sans ce champ, la vidéo ne s'affiche pas.
  6. Cliquer Publier.
  7. Vérifier sur la home : la card « Notre dernière captation » doit afficher la nouvelle vidéo.
  8. Ajouter la vidéo dans la fiche Projet du concert archivé : Articles → Tous les articles, filtrer catégorie Projets, trouver la fiche du concert (celle qu'on a créée à l'étape Archivage du concert), cliquer Modifier avec Elementor. Insérer ou mettre à jour le widget Vidéo avec l'URL YouTube qu'on vient de publier. Mettre à jour la page.
Si l'article n'apparaît pas tout de suite côté public Le site garde un cache des pages publiques. Aller sur la page concernée (home, /billetterie/, Espace choristes…), cliquer Clear Posts Cache dans le bandeau noir d'admin tout en haut, recharger. Détails : Le contenu → Les articles → Si un article ne s'affiche pas.
Si le panneau « Champs personnalisés » n'apparaît pas Cliquer sur l'icône Options (trois points en haut à droite de l'éditeur) → Préférences → onglet Panneaux → activer Champs personnalisés.

Les photos du concert

Pas-à-pas

  1. Se connecter à Drive avec orga@cug38.fr.
  2. Naviguer vers My Drive → Archives (articles, photos) → Photos du Choeur.
  3. Créer un sous-dossier au nom du concert ou de la saison (ex. 2025-26 — Materia Symphony) — c'est ce nom qui apparaîtra dans la galerie de l'Espace choristes.
  4. Déposer les photos dedans (.jpg / .png).
  5. Vérifier sur l'Espace choristes → onglet Photos. Si rien n'apparaît tout de suite, vider le cache via Clear Posts Cache dans le bandeau admin du site.

Cas 1 — Republier un article existant (le plus rapide)

Plusieurs Actualités saisonnières dorment dans la base et tournent d'une saison à l'autre. Exemple : l'article "Bonnes vacances" remis devant chaque été. Procédure :

Pas-à-pas

  6. Sidebar droite, section Publication → cliquer sur la date de publication → mettre la date d'aujourd'hui (ou la date à laquelle on veut qu'il remonte).

Les mises à jour

WordPress, le thème Hello Elementor et les extensions reçoivent des mises à jour régulières. À garder à jour pour la sécurité, mais avec un peu de prudence (les mises à jour majeures peuvent casser des choses).

Où voir les mises à jour disponibles

Tableau de bord → Mises à jour (depuis le menu de gauche). On y trouve : version de WP, version du thème, et la liste des extensions avec leur dernière version.

Ordre conseillé

  1. Sauvegarder d'abord — sur Elementor Cloud (my.elementor.com → Hosting → Backups), déclencher une sauvegarde manuelle ou vérifier qu'une sauvegarde quotidienne récente existe.
  2. Les extensions d'abord, une par une (cocher l'extension → Mettre à jour les extensions). Vérifier le site après chaque mise à jour critique (Elementor, PPWP, Google Drive Gallery).
  3. Le thème ensuite (Hello Elementor). Vérifier le rendu du header / footer.
  4. WordPress core en dernier. Vérifier le tableau de bord après.
Avant une grosse mise à jour Toujours vérifier qu'une sauvegarde récente existe (Elementor Cloud sauvegarde automatiquement chaque jour, mais on peut aussi en déclencher une à la main). Voir Sauvegardes Elementor Cloud.

Le plugin Code Snippets

Plugin Code Snippets (slug code-snippets). Permet d'ajouter du code PHP personnalisé au site sans modifier le thème ni un plugin existant. Accès dans le menu de gauche WordPress : Snippets → Tous les extraits.

Sur le site CUG, c'est la colonne vertébrale de l'approche content-driven. Chaque article qui « pilote » un emplacement Elementor (catégorie CA, Programme, Billetterie, Youtube) le fait via un ou plusieurs snippets actifs ici :

Ce qui vit dans Code Snippets

  • Enregistrement des champs personnalisés — un snippet déclare programme_youtube_playlist, programme_musescore, programme_livret_pdf, programme_mp3s, date_fin, event_map en post_meta avec show_in_rest=true. Sans ce snippet, ces champs ne sont ni éditables dans Gutenberg ni accessibles aux templates Elementor.
  • Exposition du champ Ordre (menu_order) — un snippet active page-attributes sur les articles pour faire apparaître le champ Ordre dans la sidebar Gutenberg. Essentiel pour ordonner les cartes du CA.
  • Shortcodes "lecteurs de champ" — un shortcode par custom field : [youtube_link], [programme_musescore], [programme_livret_pdf], [programme_mp3s], [programme_youtube_playlist]. Chacun renvoie la valeur du champ pour l'article courant. Utilisés dans les templates Elementor via la balise dynamique Shortcode.
  • Shortcodes "rendus prêts à l'emploi" — [programme_youtube_playlist_embed] renvoie un <iframe> YouTube complet, [event_map_embed] renvoie un <iframe> Google Maps complet (avec pin rouge garanti). Utilisés dans des widgets Shortcode Elementor.
  • Filtre de Grille de boucle Billetterie — un snippet hooke sur elementor/query/billetterie_active et limite les résultats aux articles Billetterie dont date_fin ≥ aujourd'hui. C'est lui qui fait disparaître automatiquement la card billetterie de la home quand le concert est passé.
  • Sentinelle de placeholder — [no_active_billetterie] renvoie · s'il n'y a aucune billetterie active, chaîne vide sinon. Combiné aux classes CSS cug-cond / cug-cond-host, ça fait apparaître le bloc « pas encore ouverte » à la place de la card billetterie.
  • Pattern d'affichage conditionnel — les classes CSS cug-cond-host et cug-cond combinées à un shortcode lecteur de champ permettent de cacher un conteneur entier quand le champ qu'il affiche est vide (sous-cartes Programme cachées si MP3s ou livret manquent, placeholder billetterie affiché si [no_active_billetterie] est non vide).

Le catalogue détaillé avec les sources de chaque snippet est dans Les articles → Catalogue des snippets content-driven.

Désactiver un snippet ≠ casser tout le site Désactiver un snippet casse uniquement la fonctionnalité associée (la card YouTube, la carte du lieu, le filtre billetterie, etc.). Le site reste en ligne. C'est utile pour debugger : on désactive le snippet suspect, on vérifie l'impact, on réactive.
Le pattern « shortcode → balise dynamique » Côté Elementor, la magie passe par les balises dynamiques : sur n'importe quel champ d'un widget (URL d'un bouton, URL d'une vidéo, src d'une image, classes CSS), on peut brancher une balise dynamique Shortcode avec un shortcode comme [programme_musescore]. À chaque rendu, Elementor exécute le shortcode dans le contexte de l'article courant et insère la valeur retournée. Détails et exemples : Le design → Le pattern shortcode/dynamic-tag.

L'interface de l'éditeur

Depuis 2024, Elementor utilise une interface dite Top Bar qui répartit les actions entre une barre du haut, un panneau latéral à gauche et le canvas au centre. Une fenêtre Structure flottante peut être ouverte par-dessus pour naviguer dans l'arbre de la page.

Les 4 zones de l'éditeur : la barre du haut (en noir tout en haut), le panneau latéral à gauche, le canvas au centre, et la fenêtre Structure à droite (flottante, on peut la déplacer ou la fermer).

1. La barre du haut (Top Bar)

Bandeau noir tout en haut. Concentre les actions globales :

  • À gauche : l'icône E Elementor (menu principal — Réglages de site, Notes, Centre d'aide, Quitter vers le tableau de bord), puis le nom de la page en cours d'édition, l'icône Réglages de la page et l'icône Historique.
  • Au centre : l'icône Structure qui ouvre/ferme la fenêtre Structure, et le bouton Aperçu.
  • À droite : le sélecteur Responsive (Desktop / Tablette / Mobile pour prévisualiser à chaque taille d'écran), et le bouton Publier / Mettre à jour — rose quand il y a des modifs en attente, noir quand tout est à jour côté serveur.

2. Le panneau latéral (à gauche)

Selon le contexte, deux états :

  • Liste des widgets disponibles — quand rien n'est sélectionné. On glisse-dépose un widget (Image, Titre, Bouton, Vidéo, Grille de boucle, etc.) vers le canvas.
  • Réglages du widget sélectionné — quand on a cliqué sur un élément du canvas. Trois onglets : Mise en page · Style · Avancé.

L'icône en haut à gauche du panneau (≡ retour) revient à la liste des widgets quand on est sur les réglages.

3. Le canvas (au centre)

Le rendu vivant de la page. On clique directement dessus pour sélectionner un élément. Au survol, des poignées bleues apparaissent en haut à gauche des widgets et conteneurs — utiles pour bien sélectionner un Conteneur sans cliquer accidentellement sur un widget à l'intérieur.

4. La fenêtre Structure (flottante)

Vue en arbre de toute la hiérarchie de la page : Conteneurs imbriqués et widgets. Indispensable sur les pages complexes pour :

  • Sélectionner un élément précis caché derrière un autre.
  • Réorganiser la page par drag-and-drop dans l'arbre.
  • Voir d'un coup d'œil quels éléments ont une Display Condition active (icône dédiée).

Ouverture / fermeture : icône Structure dans la barre du haut, ou raccourci Cmd/Ctrl+I.

Responsive (tablette & mobile)

Le site doit s'afficher correctement sur ordinateur, tablette et téléphone. Elementor a deux mécanismes pour ça : un sélecteur de prévisualisation (vérifier le rendu à chaque taille) et un masquage par appareil (n'afficher un élément que sur certaines tailles d'écran).

Prévisualiser le rendu sur chaque appareil

Dans la barre du haut, à droite du nom de la page, trois icônes — Desktop (ordinateur), Tablet (tablette), Mobile (téléphone). Cliquer dessus change la taille du canvas pour simuler chaque écran.

Édition de la page Le Chœur en mode Mobile. Le canvas est rétréci au format téléphone (~360 px de large) pour visualiser exactement comment la page apparaîtra. La photo de groupe passe au-dessus, le texte se replie en colonne unique. Les icônes Desktop / Tablet / Mobile sont accessibles dans la barre du haut au centre.
  1. Cliquer sur l'icône Mobile (ou Tablet) dans la barre du haut.
  2. Le canvas se rétrécit pour simuler l'écran choisi.
  3. Vérifier visuellement : les images ne dépassent pas, le texte reste lisible, les boutons sont assez gros, le menu burger fonctionne, etc.
  4. Refaire pareil avec les autres tailles puis revenir à Desktop pour continuer l'édition.

Réglages spécifiques à chaque taille

Quand on est en mode Tablette ou Mobile, la plupart des réglages d'un widget (taille de police, padding, marges, alignement, etc.) peuvent être ajustés uniquement pour cette taille d'écran — sans toucher aux autres. Une petite icône d'appareil s'affiche à côté du réglage quand il est spécifique à la taille en cours.

C'est ce qui permet, par exemple, d'avoir un titre énorme en desktop et plus modeste en mobile, ou une image en pleine largeur sur desktop et plus compacte sur téléphone, sans avoir à dupliquer les widgets.

Masquer un élément sur certains appareils

Cas d'usage typique : sur le site CUG, le header contient deux versions du menu — la version desktop horizontale et la version mobile turquoise compacte. Chacune est masquée sur les écrans qui ne la concernent pas, comme ça chaque taille n'affiche que sa version.

  1. Sélectionner le widget ou le Conteneur concerné dans le canvas (ou dans la Structure).
  2. Panneau de gauche → onglet Avancé.
  3. Déplier la section Responsive.
  4. Cocher la case correspondant à l'écran où l'élément doit disparaître : Masquer sur ordinateur, Masquer sur tablette et/ou Masquer sur Portrait mobile.
  5. Cliquer Publier / Mettre à jour.
Cases inversées : "Masquer sur…" pas "Afficher sur…" Attention au sens : cocher la case cache l'élément sur cet appareil. Pour n'afficher un élément que sur mobile, il faut cocher à la fois Masquer sur ordinateur et Masquer sur tablette.
Repérer rapidement les éléments masqués Une fois une case cochée, Elementor affiche un petit badge rose sur la poignée de l'élément dans le canvas et dans la fenêtre Structure. Ça aide à retrouver les éléments cachés sans inspecter tous les widgets un par un.

AvancéLe pattern shortcode/dynamic-tag — utilisé partout en content-driven

Quasiment tout le content-driven du site (Youtube, Programme, Billetterie) repose sur le même pattern à trois pièces. Comprendre ce pattern une fois suffit pour intervenir sur n'importe quelle catégorie content-driven. Cette section l'explique avec l'exemple Youtube — c'est l'instance la plus simple — puis liste toutes les autres utilisations du pattern sur le site.

Le pattern à trois pièces

  1. Un champ personnalisé sur l'article (post_meta WordPress) — c'est la valeur qu'on saisit dans WordPress.
  2. Un shortcode (snippet PHP dans Code Snippets) qui lit ce champ et renvoie sa valeur (ou un rendu HTML prêt à l'emploi).
  3. Une balise dynamique Shortcode sur un champ d'un widget Elementor (URL, src, classes CSS, contenu HTML…). Au rendu, Elementor exécute le shortcode dans le contexte de l'article courant et insère le résultat.

Exemple canonique — la card YouTube de la home

1. Le champ personnalisé youtube-link sur l'article

Standard WordPress (post_meta). Visible et éditable dans l'éditeur d'article, panneau Champs personnalisés. Une clé par article, valeur = URL YouTube.

2. Un shortcode [youtube_link] qui lit ce champ

Défini par un snippet PHP dans Code Snippets. Le snippet s'appelle « CUG — Shortcode [youtube_link] pour la card YouTube », est actif, en scope global.

Le snippet dans Snippets → Tous les extraits → Modifier. Le code fait juste : add_shortcode('youtube_link', fn() => esc_url(get_post_meta(get_the_ID(), 'youtube-link', true)));

3. La balise dynamique Shortcode sur le widget Vidéo du template youtube

Dans l'éditeur Elementor du template, le champ « Lien » du widget Vidéo a sa balise dynamique Shortcode branchée avec la valeur [youtube_link].

Panneau du widget Vidéo dans le template youtube. La section Réglages (visible quand la balise dynamique est active) affiche [youtube_link].

Flux complet au rendu : la Grille de boucle Youtube affiche l'article le plus récent → Elementor instancie le template youtube dans le contexte de cet article → la balise dynamique Shortcode exécute [youtube_link] → le shortcode lit get_post_meta($id, 'youtube-link') et renvoie l'URL → Elementor l'injecte dans le champ youtube_url du widget Vidéo → le lecteur YouTube charge la bonne vidéo.

Les autres applications du pattern sur le site

Catégorie Programme — fichiers de travail

Chaque article Programme a 4 champs personnalisés (programme_youtube_playlist, programme_musescore, programme_livret_pdf, programme_mp3s). Pour chacun, un shortcode du même nom renvoie la valeur. Côté template Programme (Espace choristes) :

  • Les boutons MuseScore / Livret PDF / MP3s ont leur champ URL branché en balise dynamique Shortcode avec [programme_musescore], [programme_livret_pdf], [programme_mp3s].
  • L'iframe YouTube de la playlist du semestre est rendue par un widget Shortcode Elementor qui appelle [programme_youtube_playlist_embed] — ce shortcode produit l'<iframe> complet (560×315 par défaut) à partir de l'ID stocké dans programme_youtube_playlist.

Catégorie Billetterie — carte du lieu

Chaque article Billetterie a un champ event_map (le nom du lieu, ex. « Espace Bergès Villard-Bonnot »). Dans le template de boucle Billetterie, un widget Shortcode Elementor appelle [event_map_embed] — ce shortcode lit le champ et construit l'URL maps.google.com/maps?q=…&output=embed avec pin rouge. Pas besoin de coller un iframe Google Maps à la main pour chaque concert.

Catégorie Billetterie — affichage conditionnel du placeholder

Si aucun article Billetterie n'est actif (toutes les date_fin sont passées), un Conteneur "pas encore ouverte → inscrivez-vous à la newsletter" prend la place de la card billetterie sur la home. Le pattern :

  • Le Conteneur placeholder porte la classe CSS cug-cond-host.
  • À l'intérieur, un widget Shortcode invisible (classe cug-cond) exécute [no_active_billetterie].
  • Ce shortcode renvoie · s'il n'y a aucune billetterie active, chaîne vide sinon.
  • Une règle CSS globale (.cug-cond-host:has(.cug-cond:empty) { display: none }) cache le placeholder quand la sentinelle est vide.

Le même pattern cug-cond / cug-cond-host + shortcode lecteur de champ sert à cacher les sous-cartes vides du template Programme (Musescore, MP3s, livret, YouTube).

Catégorie CA — ordre d'affichage

function importEmailsToSheet() {
  var sheet = SpreadsheetApp.openById(SHEET_ID).getSheets()[0];

  // Récupère les emails déjà présents dans le Sheet pour éviter les doublons
  var existing = new Set();
  var values = sheet.getRange('A:A').getValues();
  for (var i = 0; i < values.length; i++) {
    var v = String(values[i][0] || '').toLowerCase().trim();
    if (v) existing.add(v);
  }

  // Récupère (ou crée) le label "déjà traité"
  var label = GmailApp.getUserLabelByName(PROCESSED_LABEL);
  if (!label) label = GmailApp.createLabel(PROCESSED_LABEL);

  // Cherche les threads d'inscription non traités
  var threads = GmailApp.search(GMAIL_QUERY, 0, 100);
  var added = 0;

  for (var t = 0; t < threads.length; t++) {
    var thread = threads[t];
    var messages = thread.getMessages();
    for (var m = 0; m < messages.length; m++) {
      var body = messages[m].getPlainBody() || '';
      var match = body.match(/Email\s*:\s*([^\s<>]+@[^\s<>]+)/i);
      if (match) {
        var email = match[1].trim().toLowerCase().replace(/[.,;]+$/, '');
        if (email && !existing.has(email)) {
          sheet.appendRow([email]);
          existing.add(email);
          added++;
        }
      }
    }
    thread.addLabel(label);
  }

  console.log('Newsletter import : ' + added + ' nouvel(s) email(s) ajouté(s) · ' + threads.length + ' thread(s) traité(s).');
}

Si ça casse — playbook des incidents fréquents

Récap des symptômes qu'on peut rencontrer et le bon endroit où aller chercher dans le guide. À garder sous le coude.

🔴 Le site est totalement inaccessible

  • Domaine expiré ? Si le mail de relance OVH (tresorerie@cug38.fr, ~30 jours avant) a été ignoré, le domaine peut être suspendu. Vérifier OVH → renouvellement.
  • Hébergement Elementor expiré ? Vérifier l'état de l'abonnement Elementor Cloud sur my.elementor.com et que la carte bancaire est valide. Voir Elementor Cloud.
  • DNS modifié par erreur ? Vérifier les 2 enregistrements A/CNAME qui pointent vers Elementor dans la zone DNS OVH.

🔐 Je n'arrive plus à me connecter

  • Vérifier d'abord que tu suis bien la procédure : Se connecter à WordPress (Log in with Elementor + email/mdp LastPass, jamais Continue with Google).
  • Le bug "Something went wrong" est normal au 1er essai — fermer l'onglet, re-cliquer Log in with Elementor.
  • Si le mot de passe ne marche plus : Forgot your password? sur la page Sign in Elementor, mail de reset sur orga@cug38.fr.
  • Si le 2FA bloque : code à 6 chiffres reçu sur orga@cug38.fr, valable 10 min.

📷 Les photos de l'espace choriste n'apparaissent plus

  • Aller dans Le plugin Google Drive Gallery et vérifier que la connexion OAuth est toujours active (badge vert).
  • Si la galerie est vide : cliquer Clear Posts Cache dans le bandeau noir d'admin en haut de la page Espace Choristes côté public.
  • Si rien ne revient : voir le bloc Avancé "Migration vers un nouveau Drive" pour reconfigurer les credentials OAuth.

📧 Le Sheet Newsletter ne se met plus à jour

  • Aller dans script.google.com (compte orga@cug38.fr) → projet Gestion Newsletter → Exécutions. Si plusieurs Échec récents, regarder le détail.
  • Vérifier le déclencheur (icône horloge) — doit être actif, horaire toutes les 2 h.
  • Si le projet a disparu : voir Billetterie → Le formulaire S'inscrire (bloc Avancé déroulé) qui contient le code source à recréer.

🎬 La vidéo YouTube de la home a disparu

  • Vérifier que le snippet CUG — Shortcode [youtube_link]… est bien actif dans Snippets → Tous les extraits.
  • Vérifier que le dernier article catégorie Youtube a bien son champ personnalisé youtube-link renseigné avec une URL valide. Voir Le pattern shortcode/dynamic-tag pour la mécanique complète.

🎨 Le header / footer ne s'affiche plus correctement

  • Aller dans Header & footer via le bandeau d'admin → Modifier avec Elementor → header (ou footer).
  • Restaurer une version antérieure via l'Historique (icône horloge en haut à gauche dans Elementor) si la modif récente a cassé.
  • Si menu vide : vérifier Apparence → Menus que Menu desktop est bien attribué à l'emplacement En-tête.

🆘 J'ai cassé quelque chose et je veux revenir en arrière

  • Sur une page Elementor : icône horloge en haut à gauche de la barre d'Elementor → liste des révisions, restaurer une version.
  • Sur le site entier : Elementor Cloud sauvegarde chaque nuit. Aller sur my.elementor.com → site CUG → Hosting → Backups → restaurer une sauvegarde.
  • Sur un article (Gutenberg) : ouvrir l'article, panneau de droite → Révisions (en bas), restaurer une version.
Avant toute opération risquée Si on s'apprête à toucher : DNS OVH · mise à jour majeure WP/Elementor · suppression d'extension · changement de Drive ou de credentials OAuth → déclencher manuellement une sauvegarde via Elementor Cloud (Hosting → Backups → Create Backup) avant de commencer.
